Provably fair drawings

OnlyBall has no smart contract, so the drawing is made trustworthy by a commit-reveal scheme instead of a blockchain program. Every drawing goes through five steps, each one recorded immutably:

  1. Commit — at 00:01 ET the server generates a random 32-byte server_seed and immediately publishes server_seed_hash = SHA-256(server_seed). The real seed stays hidden.
  2. Snapshot — at 23:50 ET the ticket list is frozen: every account's ticket range, ordered by user id. snapshot_hash = SHA-256(canonical_json(snapshot)) is published.
  3. Beacon — at 00:00 ET the server reads the latest finalized Solana blockhash: a value nobody (including OnlyBall) could know before the drawing opened.
  4. Draw — winning_number = HMAC_SHA256(key=server_seed, msg=snapshot_hash + beacon) mod total_tickets. The ticket range containing that number wins.
  5. Reveal — the server_seed, beacon, snapshot and winning number are published immediately, so anyone can redo step 4 and confirm the result.

Because server_seed_hash is published before tickets close, OnlyBall cannot change the seed after seeing who bought tickets. Because the beacon is a Solana blockhash from after the snapshot, nobody could have pre-computed a favorable outcome either.
