{% comment %} Connect-wallet modal. Uses each wallet's injected browser-extension provider (window.solana for Phantom/Trust Wallet-Solana, window.ethereum for MetaMask, window.tronLink/window.tronWeb for TronLink) to sign a server-issued nonce, proving address ownership. NOTE on WalletConnect v2: the product brief also asks for Trust Wallet and MetaMask via WalletConnect v2 (for mobile wallets without a browser extension). That requires a free project ID from cloud.walletconnect.com plus the @walletconnect/sign-client relay/QR flow. Without a provisioned project ID this repo can't wire up the actual relay, so this modal talks to each wallet's injected extension provider directly -- fully functional for desktop Phantom / Trust Wallet / MetaMask today. The `connect()` dispatch below is written so a WalletConnect provider can be slotted in as a fourth branch once WALLETCONNECT_PROJECT_ID is available. {% endcomment %}
